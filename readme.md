# 실시간 공장 물류 제어 및 관제시스템 (Factory Simulator)

## 프로젝트 개요 

1. 목적: 본 프로젝트는 단순한 웹 애플리케이션 개발 보다 실제 현장에서의 작업을 소규모 형태로 시뮬레이터로 만듦으로써, 기계 내부가 어떻게 동작하는지 더 이해하기 위해 해당 애플리케이션을 만들었습니다. 실제 현장은 훨씬 더 복잡한 알고리즘 로직으로 코드로 되있겠지만, 이번 프로젝트를 만들게 되면서 알고리즘 및 동시성 제어를 다루게 되었습니다.

2. 소개: 본 시스템은 OHT 자율 주행 및 관제 시스템을 묘사한 웹 기반 시뮬레이터입니다. 사용자는 간단한 조작으로 OHT에 대해 명령을 내릴 수 있습니다. React와 스프링 부트 기반으로 제작한 소규모 프로젝트입니다.

## 핵심 로직 

### 1. 다익스트라 알고리즘 
- 최단 거리라도 개체가 많이 있으면 병목 현상이 나타납니다. 이로 인해 특정 Edge에 두 대 이상 진입하게할 경우, 해당 구간의 가중치에 페널티를 부여합니다. 뒤따라오는 개체들은 실시간으로 다익스트라 알고리즘을 재연산하여 다른 우회로를 찾아 트래픽을 분산시킵니다.
해당 알고리즘은 매우 정교한 계산을 하기 때문에 속도가 느립니다. -> A*알고리즘 활용 가능


### 2. 충돌 방지 시스템과 교차로 상호배제
- 다수의 개체가 동일한 블록을 지날 때 발생하면 데드락이 걸리게 되거나 물리적 충돌이 발생합니다. 그렇기 때문에 각 개체마다 일정 간격으로 유지하여 움직이게 하기 위해 특정 영역에 다른 개체가 감지되면 Wait를 작동하게 하여 마치 꼬리 물듯 구현하였습니다. 교차로에서도 충돌이 일어나지 않게 특정 영역에 감시하면 락을 걸며 개체끼리 충돌하게 되는 것을 원천적으로 차단하였습니다. 해당 맵은 너무 짧은 트랙을가졌기도 하고, 데드락을 방지하기 위해 단방향으로 설계하였습니다. 


### 3. 가장 가까운 배차 시스템
- 디스패치 명령 시, 전체 대기 차량 중 출발지와의 유클리드 거리가 가장 짧은 개체를 수학적으로 계산하여 즉시 호출합니다. 


### 4. 자율 배터리 관리
- 개체는 이동 시 배터리가 소모되며, 잔량이 20퍼 이하로 떨어지게 되면 스스로 충전소로 목적지로 변경하여 완충 후 복귀하는 상태머신을 구현하였습니다. 하지만 이 충전소 위치 때문에 빨간 신호에 걸린 것 마냥 모두 대기 상태가 되버리는 치명적인 상황이 연출됩니다. 


## 시스템 아키텍처

1. Backend : Spring Boot | 0.1초 단위로 총 15대의 개체 좌표 및 상태를 연산하는 물리 엔진 및 라우팅 지시 역할
2. Message Broker: Redis | 대규모 실시간 좌표 데이터를 WebSocket으로 브로드캐스팅하기 위한 파이프라인
3. Frontend: React(Canvas API 활용) | 60FPS로 다수의 객체를 렌더링하여, 클릭 이벤트를 통한 양방향 제어가 가능 
4. Data Pipeline: H2 DB | 실시간 서비스 성능 저하를 막기 위해 10초 주기로 상태 로그를 비동기



## 핵심 코드
1. App.js(React) : 렌더링, 개체 상태 데이터를 실시간으로 수신, 양방향 제어
2. OhtSimulator.java: 0.1초마다 개체 좌표, 배터리, 상태를 계산. 다익스트라 우회 경로를 탐색, 충돌 방지
3. OhtController.java: 프론트엔드의 클릭 및 명령을 백엔드 엔진으로 전달하는 역할
4. OhtLogService.java : 메모리에 로그를 모아뒀다가 10초에 한 번씩 백그라운드에서 몰아서 DB에 저장하는 파이프라인
5. WebSocketConfig.java, RedisConfig.java : 서버와 클라이언트 간의 실시간 고속도로(WebSocket)를 뚫고, 그 고속도로에 데이터를 공급해 줄 물류 창고(Redis)를 세팅하는 환경 설정 파일

## 사용
1. java version : 21 , Node.js : 18, Docker Desktop(Redis 실행용)
2. 인프라 환경 세팅
- docker run -d --name oht-redis -p 6379:6379 redis 
3.  스프링부트 실행
4. React 실행
cd frontend / npm install / npm start
